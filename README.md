## Overview

Express-BookApp is a backend REST API that provides full **Create, Read, Update, and Delete (CRUD)** operations for a book management system. It is written in TypeScript with strict typing, compiled to ES Modules, and served through Express.js running on port **80**.

The project is structured around clean separation of concerns — routing, data, types, and infrastructure each live in their own layer — making it straightforward to extend or integrate with a database in the future.

---

## Project Structure

```
Express-BookApp/
├── data/              # Static or seed data for books
├── dist/              # Compiled JavaScript output (generated by tsc)
├── infrastructure/    # App-level configuration and infrastructure setup
├── routers/
│   └── book.ts        # All /book route handlers (CRUD)
├── types/             # TypeScript interfaces and type definitions
├── index.ts           # Entry point — Express app initialization
├── package.json
└── tsconfig.json
```

## Getting Started

### Prerequisites

- Node.js ≥ 18
- npm

### Installation

```bash
git clone https://github.com/SarahAbuirmeileh/Express-BookApp.git
cd Express-BookApp
npm install
```

### Running in Development

The dev setup compiles TypeScript in watch mode and restarts the server automatically on changes via `nodemon`:

```bash
npm run dev
```

This runs two parallel processes:
- `tsc -w` — watches and recompiles `.ts` files into `dist/`
- `nodemon ./dist` — restarts the server whenever compiled output changes

The server starts on **port 80**.

To build once without watching:

```bash
npm run build-tsc
```

## API Endpoints

The app mounts all book-related routes under the `/book` prefix.

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | Health check — returns "Hello world" |
| GET | `/health` | Returns HTTP 200 (liveness probe) |
| GET | `/book` | Retrieve all books |
| GET | `/book/:id` | Retrieve a specific book by ID |
| POST | `/book` | Create a new book |
| PUT | `/book/:id` | Update an existing book |
| DELETE | `/book/:id` | Delete a book |

> Any unmatched route returns `404 — Wrong URL :(`.
